KR260を購入

2026-07-01

以前、業務で評価したことのあるKR260を個人的に購入した。なぜ購入したかというと、前からFPGAの開発環境がほしかったことと、NASとして10年以上使用していたサーバーをいい加減リプレースしたかったこと、USBが4ポートついているので外付けでRAID1が十分組めるので丁度良いこと、さらに最近のメモリ不足や円安で価格が少しずつ上がっていたこと、などが重なり決断した。Xilinx向けのArm Cortex-m1やRISC-Vのソフトコアも使えるので、いろいろと遊べそうである。

digikeyで購入し、2日程度で到着した。さっそく、Ubuntu Server 24.04 LTSをダウンロードしてSDカードに書き込む。通常このままだと平文でデータが書き込まれてしまうので、rootfsは暗号化して運用することにする。(暗号化についてはラズパイ５のディスク暗号化と手動起動を参考)

注意 ブロックデバイスは自環境に合わせること

pc: $ xzcat iot-limerick-kria-classic-server-2404-classic-24.04-x07-20250423.img.xz | sudo of=/dev/sdb status=progress
# ここでパーティションツールを使い縮小する。ここでは4 GiBで収まった。
pc: $ sudo dd if=/dev/sdb2 of=kria-rootfs.img status=progress
# ここでパーティションを最大にする。
pc: $ sudo cryptsetup luksFormat /dev/sdb2
pc: $ sudo cryptsetup open /dev/sdb2 luks
pc: $ sudo dd if=kria-rootfs.img of=/dev/mapper/luks status=progress

SDカードをKR260に挿してブートする。初回起動時、復号できないのでbusyboxのシェルが表示される。手動で復号して起動する。

(initramfs) cryptsetup open /dev/sda2 luks
(initramfs) exit

起動後、/etc/crypttabに復号できるように設定する。

+luks /dev/disk/by-uuid/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx none luks,discard

initramfsを更新する。

raspi: $ sudo update-initramfs -u -k all

再起動後、ブート中に暗号化時に設定したパスワードが聞かれるので手動で入力することで起動した。
