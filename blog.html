ラズパイ５のディスク暗号化と自動復号

2026-06-20

前回「ラズパイ５のディスク暗号化と手動起動」はパスフレーズをキーボードから手動で入力することで起動できるようにした。しかし、これでは毎回パスの入力を行う手間がかかることから、今回はキーファイルを読み込ませることで自動で起動するように行った。USBフラッシュメモリに保存したキーファイルで復号することで、物理キーとして扱えるようにした。

注意 ブロックデバイスは自環境に合わせること

キーファイルをUSBフラッシュメモリに作成して、新しい鍵として登録する。

raspi: $ sudo mkdir -p /mnt/usbflash
raspi: $ sudo mount /dev/sda1 /mnt/usbflash
raspi: $ sudo dd if=/dev/urandom of=/mnt/usbflash/keyfile bs=512 count=4 iflag=fullblock
raspi: $ sudo cryptsetup luksAddKey /dev/mmcblk0p2 /mnt/usbflash/keyfile

ブート中に復号するスクリプト/etc/initramfs-tools/scripts/init-premount/luks-mount-scriptを作成した。

#!/bin/sh

PREREQ=""
prereqs() {
  echo "$PREREQ"
}

case $! in
  prereqs)
    prereqs
    exit 0
    ;;
esac

mkdir -p "${rootmnt}/usbflash"
mount /dev/disk/by-uuid/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx "${rootmnt}/usbflash"
cryptsetup open /dev/disk/by-uuid/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx luks --key-file "${rootmnt}/usbflash/keyfile"

exit 0

UUIDはlsblkで確認した。

raspi: $ lsblk -f
NAME        FSTYPE      FSVER LABEL  UUID                                 FSAVAIL FSUSE% MOUNTPOINTS
loop0       swap        1                                                                
sda                                                                                      
└─sda1      ext4        1.0          xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx                
mmcblk0                                                                                  
├─mmcblk0p1 vfat        FAT32 bootfs xxxx-xxxx                               389M    23% /boot/firmware
└─mmcblk0p2 crypto_LUKS 2            xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx                
  └─luks    ext4        1.0   rootfs xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  195.2G    10% /
zram0       swap        1     zram0  xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx                [SWAP]

initramfsを更新する。

raspi: $ sudo update-initramfs -u -k alld

これで自動で復号されて起動するはずだが、USBフラッシュメモリの認識がスクリプトの実行よりも遅いため失敗することがあった。そのためcmdline.txtに遅延rootdelay=2を追加した。

-console=serial0,115200 console=tty1 root=/dev/mapper/luks rootfstype=ext4 fsck.repair=yes rootwait cfg80211.ieee80211_regdom=JP
+console=serial0,115200 console=tty1 root=/dev/mapper/luks rootfstype=ext4 fsck.repair=yes rootwait cfg80211.ieee80211_regdom=JP rootdelay=2

再起動後、自動で起動することが確認できた。USBフラッシュメモリを外すと手動起動と同じ動作となった。

ラズパイ５のディスク暗号化と手動復号

2026-06-16

Rasbperry Pi Compute Module 5(CM5)を勉強のために購入したものの、しばらく積んだ状態にしていた。時間ができたので活用できるようにセットアップをしてみた。

Raspberry Piは、公式のイメージのまま使用すると特に暗号化などがされていないので、盗難や紛失時に情報が流出する恐れがある。今回はディスクを暗号化し、起動時にパスワードを入力することで起動できるようにした。

はじめに公式ドキュメントのインストールに従って起動用SDカードを用意する。ここではRaspberry Pi OS Lite (64-bit)を選択し、ddコマンドで書き込みした。

注意 ブロックデバイスは自環境に合わせること

pc: $ xzcat 2026-04-21-raspios-trixie-arm64-lite.img.xz | sudo dd of=/dev/mmcblk0 status=progress

ラズパイを起動して必要なパッケージをインストールする。

raspi: $ apt update
raspi: $ apt full-upgrade
raspi: $ apt install cryptsetup cryptsetup-initramfs

/etc/crypttabに起動時に復号できるように設定する。

+luks /dev/mmcblk0p2 none luks,discard

/etc/fstabに復号されたデバイスブロックをrootfsとしてマウント設定する。

-/dev/mmcblk0p2 /               ext4    defaults,noatime  0       1
+/dev/mapper/luks  /               ext4    defaults,noatime  0       1

/boot/firmware/cmdline.txtのrootを複号したブロックデバイスに変更する。

-console=serial0,115200 console=tty1 root=/dev/mmcblk0p2 rootfstype=ext4 fsck.repair=yes rootwait cfg80211.ieee80211_regdom=JP
+console=serial0,115200 console=tty1 root=/dev/mapper/luks rootfstype=ext4 fsck.repair=yes rootwait cfg80211.ieee80211_regdom=JP

/etc/initramfs-tools/initramfs.confを編集して、initramfsを更新する

-MODULES=dep
+MODULES=most

raspi: $ sudo update-initramfs -u -k all

作業PCにSDカードを戻してパーティションを縮小する。パーティションサイズの縮小には、KDEパーティションマネージャを使用した。GNOMEであればGPartedが使えるだろう。縮小後、rootfsをコピーする。

pc: $ sudo dd if=/dev/mmcblk0p2 of=raspi-rootfs.img status=progress

コピーサイズは約8GBだった。コピー後、パーティションを拡張して最大にする。 次にLUKSでrootfsパーティションを暗号化し、コピーしたデータを戻す。

pc: $ sudo dd if=/dev/urandom of=/dev/mmcblk0p2 status=progress
pc: $ sudo cryptsetup luksFormat /dev/mmcblk0p2
pc: $ sudo cryptsetup open /dev/mmcblk0p2 luks
pc: $ sudo dd if=raspi-rootfs.img of=/dev/mapper/luks status=progress
pc: $ sudo cryptsetup close luks

SDカードをラズパイに挿してブートする。初回起動時、復号できないのでbusyboxのシェルが表示される。手動で復号して起動する。

(initramfs) cryptsetup open /dev/mmcblkp02 luks
(initramfs) exit

ブート後、initramfsを更新する

raspi: $ sudo update-initramfs -u -k all

再起動後、ブート中に暗号化時に設定したパスワードが聞かれるので手動で入力することで起動した。
